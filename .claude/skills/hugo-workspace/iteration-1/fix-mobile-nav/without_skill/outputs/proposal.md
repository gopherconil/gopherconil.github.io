# Proposal: Fix Mobile Hamburger Menu Not Closing on Nav Link Click

## Root Cause

The mobile menu JavaScript is **never loaded** on the current site. In `layouts/index.html` (lines 48-53), the `<script src="/js/main.js">` tag is conditionally included only when both `Address` and `GoogleMapsKey` site parameters are set:

```html
{{ if isset .Site.Params "address" }}
{{ with .Site.Params.GoogleMapsKey }}
<script src="https://maps.google.com/maps/api/js?key={{ . }}"></script>
<script src="/js/main.js"></script>
{{ end }}
{{ end }}
```

In `config.yml`, both of these parameters are commented out (lines 17 and 22):

```yaml
# Address: "Expo - Bitan 10"
# GoogleMapsKey: "my-secret-maps-key"
```

Because `main.js` is never loaded, **none** of the mobile menu JavaScript runs -- the hamburger toggle doesn't work, nav link clicks don't close the menu, and smooth scrolling is disabled.

The code inside `main.js` (lines 155-159) already contains the correct logic to close the mobile menu when a nav link is clicked. The problem is purely that the script is not loaded.

## Proposed Fix

**File to edit:** `/Users/yonidavidson/dev/private/gopherconil.github.io/layouts/index.html`

**Change:** Move the `main.js` script tag outside the Google Maps conditional block so it always loads. Keep the Google Maps API script inside the conditional. Also, refactor `main.js` so that the Google Maps initialization does not crash when the Maps API is unavailable.

### Step 1: Edit `layouts/index.html` (lines 46-53)

Replace:

```html
    <script src="https://ajax.googleapis.com/ajax/libs/jquery/3.7.1/jquery.min.js"></script>
    <script>window.jQuery || document.write('<script src="https://code.jquery.com/jquery-3.7.1.min.js"><\/script>')</script>
    {{ if isset .Site.Params "address" }}
    {{ with .Site.Params.GoogleMapsKey }}
    <script src="https://maps.google.com/maps/api/js?key={{ . }}"></script>
    <script src="/js/main.js"></script>
    {{ end }}
    {{ end }}
```

With:

```html
    <script src="https://ajax.googleapis.com/ajax/libs/jquery/3.7.1/jquery.min.js"></script>
    <script>window.jQuery || document.write('<script src="https://code.jquery.com/jquery-3.7.1.min.js"><\/script>')</script>
    {{ if isset .Site.Params "address" }}
    {{ with .Site.Params.GoogleMapsKey }}
    <script src="https://maps.google.com/maps/api/js?key={{ . }}"></script>
    {{ end }}
    {{ end }}
    <script src="/js/main.js"></script>
```

### Step 2: Edit `themes/hugo-conference/static/js/main.js` (and `static/js/main.js` and `docs/js/main.js`)

Guard the Google Maps initialization so it doesn't crash when the Maps API is not loaded. Change `conf.init` (lines 7-12) from:

```js
conf.init = function () {
    conf.map.init($('#map-canvas'));
    conf.menu.init();
    conf.mobileMenu.init();
    conf.scrollSpy.init();
};
```

To:

```js
conf.init = function () {
    if (typeof google !== 'undefined' && google.maps) {
        conf.map.init($('#map-canvas'));
    }
    conf.menu.init();
    conf.mobileMenu.init();
    conf.scrollSpy.init();
};
```

## Files Affected

1. **`/layouts/index.html`** -- Move `main.js` script tag outside the Google Maps conditional
2. **`/themes/hugo-conference/static/js/main.js`** -- Guard `conf.map.init()` call
3. **`/static/js/main.js`** -- Same guard (this file overrides the theme copy)
4. **`/docs/js/main.js`** -- Same guard (this is the built output; will be regenerated by Hugo, but should also be updated directly if not rebuilding)

## Why This Works

- The mobile menu toggle, smooth scroll, scroll spy, and nav-link-closes-menu functionality all live in `main.js` and are independent of Google Maps.
- The existing code at `main.js` lines 155-159 already correctly closes the menu on nav link click by removing the `active` class from both the toggle button and the menu, and removing `menu-open` from the body.
- The only issue was that the script was never loaded. Moving it outside the conditional fixes that.
- Guarding `conf.map.init()` prevents a runtime error (`google is not defined`) when the Maps API script is not present.

## Impact

- Mobile hamburger menu will open/close correctly
- Clicking any nav link in the mobile menu will close the menu
- Smooth scrolling to sections will work
- Scroll spy (active nav link highlighting) will work
- No regression: when Google Maps params are re-enabled, the map will continue to work as before
